## Automatic Logging

bleed automatically sends a log to **#log** for every formal action taken via the bot. Each entry includes the action, target user, moderator, reason, and timestamp.

This is why **every command must include a reason.** A log with no reason tells us nothing.

| Action               | Auto-logged?  |
| -------------------- | ------------- |
| `,warn`              | ✅             |
| `,timeout` / `,mute` | ✅             |
| `,jail` / `,unjail`  | ✅             |
| `,kick`              | ✅             |
| `,ban` / `,unban`    | ✅             |
| `,purge`             | ✅             |
| Verbal warning       | ❌ Manual only |

---

## Reviewing a Member's History

```bash
,history @user
```

Always check this before taking action. A clean record warrants more leniency; a long history warrants going straight to the next tier.

---

<Tip>
  When in doubt about what tier a member is on for a specific offense, check their history for past punishments with similar reasons.
